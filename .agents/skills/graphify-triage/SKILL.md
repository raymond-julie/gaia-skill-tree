# Graphify Triage

## Overview
Graphify Triage is a fused skill that bridges static architectural analysis with project management. It uses the `graphify` skill to map the codebase and the GitHub CLI (`gh`) to convert architectural debt into tracked issues.

## Dependencies
- **graphify**: Required for AST extraction and graph generation.
- **github-cli**: Required for issue creation and label management.

## Quick Start
```bash
# 1. Audit the current directory and preview proposed issues
uv run ~/.agents/skills/graphify-triage/scripts/triage.py audit --path . --output payloads.json

# 2. Sync findings to GitHub
uv run ~/.agents/skills/graphify-triage/scripts/triage.py sync --input payloads.json
```

## Utility Scripts
The `triage.py` script orchestrates the pipeline:

### `audit`
Runs the graphify pipeline and parses `graphify-out/GRAPH_REPORT.md` to identify:
- **God Nodes**: High-centrality components that need decoupling.
- **Audit Questions**: Structural patterns that require manual review.
Saves these as a JSON payload for review or sync.

### `sync`
Pushes the JSON payloads generated by `audit` to GitHub.
- Fetches labels and tags issues appropriately (`architecture`, `audit`).
- Automatically handles repo detection via `gh`.

## Workflow
1. **Initialize**: The agent ensures `graphify` is installed and the project has a `graphify-out/` directory.
2. **Scan**: Run `audit`. This ensures the graph is updated with `--update` and identifies new weaknesses.
3. **Review**: The agent presents the `payloads.json` summary to the user.
4. **Push**: Run `sync` to log the issues.
5. **Log**: (Gaia specific) The agent generates a Gaia intake node for this fused skill.

## Common Mistakes
- **No Auth**: Running `sync` before authenticating with `gh auth login`.
- **Stale Graph**: Forgetting to run `audit` after major code changes.
- **Missing Labels**: If the target repo doesn't have `architecture` or `audit` labels, `gh` may prompt for creation (interactive mode).
