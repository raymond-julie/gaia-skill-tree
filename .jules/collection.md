## 2026-06-24 - Evidence Collection Strategy: Libraries and Security Communities

**Learning:** When collecting evidence for generic or proposed science skills (e.g., in GAI-232), mapping them to their primary implementing libraries (like DeepChem for drug discovery or Scanpy for genomics) is essential for high-trust verification. High-signal community sources like `linux.do` are critical for security-related agent skills (`reverse-skill`) which may lack traditional ArXiv validation.

**Action:**
1. **Source Mapping:** For foundational AI capabilities, always link to the primary GitHub repository to establish `github-stars-own` and seek a peer-reviewed system demonstration paper (`peer-review`).
2. **Community Signals:** Monitor niche community forums (e.g., `linux.do`) for `social-signal` markers when dealing with emerging security orchestration tools.
3. **Negative Findings:** Document "non-related" evidence (e.g., general RE papers for a specific routing framework) in issue comments to prevent redundant future scrapers from following low-signal paths.
