This PR looks great, but there are a few issues causing the CI to fail that need to be addressed before it can be merged:

1. **`auto-merge-small-prs.yml` Regex Bug:** The change to the `CRITICAL_FILES` grep regex `^(\.github/|pyproject\.toml|package\.json|package-lock\.json)` uses `^` which anchors all the patterns to the start of the file path. This accidentally stops blocking changes to the manifest files in subdirectories (like `packages/cli-npm/package.json` and `packages/mcp/package.json`), allowing them to be auto-merged. If you want to continue blocking all package manifests, you should separate the anchors, e.g., `grep -E '(^\.github/|^pyproject\.toml|package\.json|package-lock\.json)'`.

This string was updated in this PR. You'll need to update the test in `tests/test_workflows.py` to match the new string or logic.

3. **Branch Scope Check Failure:** The `Branch Scope Check` action is failing because your branch is named `infra/update-branch-scope-conventions`, but you've included changes to `src/`, `packages/`, and `registry/` (due to the version bumps and the new `--sync` flag). `infra/` branches are not allowed to touch these files. To fix this, you can either separate the `gaia release --sync` feature into a different branch (like `cli/*` or `dev/*`), or ask a maintainer to add the `skip-scope-check` label to this PR so it can bypass the check.
