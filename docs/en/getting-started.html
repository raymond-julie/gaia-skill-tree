Getting Started

Install, scan, and claim your first skill.

This guide walks you through installing the Gaia CLI, initialising your profile, running a repo scan, and promoting your first skill — from zero to a ranked entry in the registry in under 10 minutes.

Prerequisites

Gaia's CLI is a Python package. You need:

  • Python 3.10 or newer
  • pip (comes with Python) or pipx for isolated installs
  • A Git repository — gaia scan and gaia push require one
ℹ

Not in a repo? You can still run gaia init and gaia scan outside a Git repo for exploration, but gaia push will exit with an error because it needs a repo URL to build the skill batch. See the non-repo environments section below.

Installation

Option A — install script (recommended)

shell
curl -fsSL https://gaia.tiongson.co/install.sh | sh

Prefers pipx if available, otherwise falls back to pip install --user and prints a PATH hint if needed. Requires Python 3.8+.

Option B — pipx (isolated global install)

shell
# Install pipx if you don't have it
pip install pipx

# Install gaia in its own isolated environment
pipx install gaia-cli

Option C — pip

shell
pip install gaia-cli

Installs the gaia binary into the active Python environment. Use a virtualenv to avoid touching your system Python.

Option D — editable install from source

shell
git clone https://github.com/mbtiongson1/gaia-skill-tree.git
cd gaia-skill-tree
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev,embeddings]"

Verify the install

shell
gaia --version
# gaia 4.7.0

Initialise your profile

Navigate to your agent repo, then run gaia init with your GitHub handle:

shell
cd my-agent-repo
gaia init --user <your-github-handle>

This creates a user profile at skill-trees/<your-handle>/skill-tree.json in the registry (if you're working inside the Gaia repo itself) or initialises a local config so gaia scan knows where to write results.

✦

Your handle is the string that will appear in honor red next to any Named Skill you claim. Use your actual GitHub username — it's permanent in the registry once a skill is named to you.

Check who you are

Run gaia whoami at any time to confirm your authorization path:

shell
gaia whoami
# User:  your-handle
# Via:   bootstrap   (or verifier / override / denied)

The via field tells you which authorization path applies. bootstrap means no 4★ Verifiers exist yet; all mutations are allowed. verifier means you hold a Named Skill at 4★ or above.

Scan your repo

gaia scan walks your repo looking for skill evidence — SKILL.md files, .claude/skills/ directories, agent configurations, and more. It then produces a list of promotion candidates: skills detected at a new star level.

shell
gaia scan
# Scanning repo: my-agent-repo
# Found 3 candidates → generated-output/promotion-candidates.json
⚠

Promotion candidates expire after 24 hours. Run gaia promote in the same session, or re-run gaia scan if the file has gone stale.

What the scanner looks for

In order of precedence, the scanner inspects:

  • SKILL.md — the canonical skill definition file at any nesting level
  • .claude/skills/ and .agents/skills/ — local agent skill directories
  • .cursor/rules/, .github/copilot-instructions.md, .windsurfrules — tool-specific instruction files
  • Custom skill paths configured in .gaia/config.toml

Each detected skill is matched against the canonical registry using semantic similarity. If a match is found, the skill inherits the canonical node's lineage and graph placement. Unmatched skills appear as new starless generics.

Promote a skill

After a scan, promote a candidate to your skill tree:

shell
# List candidates from the last scan
gaia scan

# Promote a specific skill (use the /slug from the scan output)
gaia promote /tool-use

# Or promote interactively — the CLI shows a ranked selection menu
gaia promote

A successful promotion writes a ranked entry to your skill-trees/<handle>/skill-tree.json and appends a rank_up event to the timeline array. The timeline is the auditable history of your progression — it cannot be hand-edited without leaving a gap in the record.

✦

Skill IDs in scan output are slash-prefixed slugs like /tool-use or /code-generation. Pass that exact string to gaia promote.

View your skill tree

shell
# Your local-first tree — shows only your skills
gaia tree

# Full canonical registry tree (all contributors, all skills)
gaia tree --canon

The default gaia tree view is local-first: it shows your own unlocked skills at their current star levels, with Named forms when they exist. Pass --canon to see the canonical registry view instead.

Visual graph

shell
# Opens a 3D skill graph in your browser
gaia graph

The graph renders the dependency DAG: nodes are skills, edges are prerequisite relationships, and tier color (sky-blue / purple / deep-violet / amber) distinguishes Basic / Extra / Unique / Ultimate.

Submit for intake

When you're ready to submit your skills for public registry review, run gaia push. This opens a GitHub issue with a structured skill batch for maintainer review.

shell
# Dry-run first — see what would be submitted without opening an issue
gaia push --dry-run

# Submit for real
gaia push
⚠

Always dry-run first. gaia push opens a public GitHub issue. The --dry-run flag lets you preview the batch JSON without creating the issue.

Once submitted, maintainers review each proposed skill using the Reviewer Checklist in the issue body. Accepted skills are promoted into registry/gaia.json in a separate review/meta/… PR.

Core concepts

The four tiers

Every skill in the registry belongs to exactly one tier:

Tier Symbol Description
○ Basic ○ A primitive, indivisible capability — the genome of every agent.
◇ Extra ◇ Emerges when two or more lower-tier skills fuse.
◉ Unique ◉ A Basic Skill that reached elite mastery with no fusion path forward.
◆ Ultimate ◆ High-complexity emergent capability found in fewer than 1% of agents.

Stars (maturity)

Stars measure verified maturity on a 0★ → 6★ axis, derived from evidence — never declared. The rank names at each star level are:

  • 0★ — Unawakened: detected but not yet evidenced
  • 1★ — Awakened: first sighting, Class C evidence
  • 2★ — Named: claimed by an Origin Contributor, reproducible
  • 3★ — Evolved: documented, Class B evidence
  • 4★ — Hardened: battle-tested, multi-source evidence
  • 5★ — Transcendent: peer-reviewed, Class A evidence
  • 6★ — Transcendent ★ (Apex): the rarest, most verified tier
ℹ

Tier ≠ Stars. Tier is the taxonomy (Basic / Extra / Unique / Ultimate). Stars are the maturity level. A Basic Skill can reach 6★; an Ultimate starts at 0★. Never use "tier" to mean "star level."

Local-first design

Gaia is built local-first: every command defaults to your context — your unlocked skills, your detected skills, your Named forms. Pass --canon to any command to switch to the canonical registry view.

This means:

  • gaia tree shows your tree
  • gaia stats shows your stats
  • gaia graph generates a graph of your project-local skills

Named Skills and origin

A Named Skill is a canonical skill claimed by a real contributor with Class C evidence or better. At the moment of naming, the skill reaches 2★. Your handle appears in honor red on the registry page as the Origin Contributor.

Named Skills are structured as Markdown files at registry/named/<handle>/<skill-slug>.md. The CLI handles all writes — never edit these files by hand.

Non-repo environments

Some AI agent environments (like Hermes, OpenClaw, or custom operator setups) don't run inside a Git repository. gaia init and gaia scan work fine there — you can build a local skill tree and visualise your graph. But gaia push will exit with an error because it needs a repo URL to build the intake batch.

⚠

If you run gaia init and gaia scan in a non-repo environment and then try gaia push, the CLI will warn you that intake is not supported outside a Git repo. This is by design — skill batches require a source URL for maintainer verification.

What you can do outside a repo

  1. Build a local skill tree

    Run gaia scan anywhere — it will detect skills in your local file system and write results to generated-output/promotion-candidates.json.

  2. Visualise the graph

    Run gaia graph to open a browser-based 3D graph of your locally detected skills overlaid on the canonical registry.

  3. View your tree

    Run gaia tree to see your current promotion state in the terminal.

  4. Contribute via a public repo

    To use gaia push, publish your skill files to a public GitHub repo and run the CLI from inside that repo's working directory.
