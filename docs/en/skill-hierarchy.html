Skill Hierarchy

Gaia organizes capabilities along two orthogonal axes: tier (what kind of skill it is) and stars (how mature the evidence is). Understanding both is essential for reading the tree, interpreting the graph, and planning your next fusion.

Overview

Every skill in the Gaia registry lives at the intersection of a tier and a star value. These are independent axes — a Basic skill can reach 6★ just as an Ultimate can sit at 0★. Neither axis implies the other.

A third axis — rarity — existed in earlier schema versions but has been fully removed. If you encounter old documentation referencing common/uncommon/rare/epic/legendary, treat it as stale. All current signal lives in tier and stars.

Tier axis — what kind
○ Basic primitive, indivisible
◇ Extra emergent via fusion
◉ Unique elite Basic, no fusion path
◆ Ultimate apex complexity, <1% of agents
Stars axis — how mature
0★ Unawakened — no evidence
1★ Awakened — first sighting
2★ Named — claimed + Class C
3★ Evolved — Class B evidence
4★ Hardened — verified + Class A
5★ Transcendent — peer-reviewed
6★ Transcendent ★ — Apex

The four tiers

Tier is a structural category describing how a skill came to exist and whether it can fuse further. It is determined at curation time and does not change with evidence accumulation.

○
Basic Skill
Section header: Basics
A primitive, indivisible capability — the genome of every agent. Basic skills are the atoms from which all higher tiers are built. They can fuse with other Basics to produce Extras or Ultimates. Most of the registry is made up of Basic skills.
◇
Extra Skill
Section header: Extras
A capability that emerges when two or more lower-tier skills fuse. Extra skills can themselves fuse with other Extras to produce more complex Extras or Ultimates. Never created directly — always the product of a fusion path.
◉
Unique Skill
Section header: Uniques
A graph-isolated Basic skill that reached elite mastery through depth alone — no fusion path forward exists for it. Requires 4★ Hardened rank and an existing named implementation to promote to Unique via gaia promote --unique.
◆
Ultimate Skill
Section header: Ultimates
A high-complexity emergent capability found in fewer than 1% of agents — the apex tier. Ultimates require broad prerequisite coverage across multiple sub-trees, making them the hardest skills to legitimately demonstrate and the most meaningful to hold.
Tier is structural, not aspirational. A Basic skill is not "worse" than an Ultimate — it is a different kind of capability. Some of the most useful and widely-held skills in the registry are Basic. Stars measure depth; tier measures shape.

The stars axis

Stars measure how thoroughly a skill's real-world existence has been demonstrated and peer-reviewed. They are derived from evidence, never declared. An agent cannot claim to be "5-star at web search" by self-reporting — stars accumulate through a sequence of evidence submissions and Verifier reviews.

The axis runs from 0★ (no evidence) to 6★ (the Apex). Rank names are the canonical labels for specific star values; they are only valid when paired with the name — "the Hardened rank", never just "rank 4."

Rank names

Stars Rank name Notes
0★ Unawakened No evidence. Also rendered as Pure in compact skill pills.
1★ Awakened First sighting — a single Class C evidence entry exists.
2★ Named Skill has been claimed by a contributor with Class C evidence or better. Origin Contributor name attached.
3★ Evolved Class B evidence — reproducible, documented demonstration.
4★ Hardened Class A evidence — battle-tested. Threshold for Verifier authorization.
5★ Transcendent Multiple peer-reviewed Class A entries.
6★ Transcendent ★ The Apex. Reserved for skills with exhaustive, community-validated evidence across multiple independent sources.
Do not confuse tier and stars. "Tier" (Basic/Extra/Unique/Ultimate) describes the structural category. "Stars" (0★–6★) describes the evidence depth. "Rank" is only the name for a specific star value — never use "rank" as a synonym for the full stars axis.

Evidence classes

Evidence classes grade the quality of a real-world demonstration. Every star advance above 0★ requires at least one evidence entry meeting the threshold for that rank. Evidence is attached by Verifiers using gaia dev evidence.

C — Bronze
First sighting. A single, credible observation of the skill in a real repository or agent context. No reproducibility required. Sufficient for the 1★ Awakened rank. All Named Skills (2★) require at least Class C.
B — Silver
Reproducible and documented. A demonstration that another evaluator can independently verify. Typically a public GitHub repo with a SKILL.md file and test coverage. Enables the 3★ Evolved rank.
A — Gold
Battle-tested and peer-reviewed. Published benchmarks, academic papers (arXiv or equivalent), or broad community verification. Required for the 4★ Hardened rank and above. Grants Verifier authorization to the holder.
attach evidence via CLI
# Class C — first sighting (requires Verifier auth)
gaia dev evidence web-search https://github.com/you/your-agent --class C

# Class B — reproducible repo with SKILL.md
gaia dev evidence web-search https://github.com/you/your-agent --class B \
  --notes "Tested across 50 queries, results logged in SKILL.md"

# Class A — peer-reviewed paper
gaia dev evidence web-search https://arxiv.org/abs/2401.00000 --class A

Fusion

Fusion is the act of combining two or more skills into a single higher-complexity skill. It is the mechanism by which Basic skills produce Extra skills, and Extra skills produce Ultimates. The verb is always fuse — never merge, combine, or compose.

A fusion candidate appears in your tree when you already hold all of a target skill's prerequisite components. Confirm it with gaia fuse. The fused skill starts at a star value equal to the highest star among its components, minimum 1★.

Fusion paths
○ web-search + ○ parse-html + ○ extract-entities → ◇ web-scraping
Three Basic skills fuse into one Extra skill.
◇ web-scraping + ◇ long-context-synthesis + ◇ tool-calling → ◆ autonomous-research
Extra skills fuse into an Ultimate. Prerequisite chains can run deep.
○ code-review → 4★ Hardened + graph-isolated → ◉ code-review (Unique)
A Basic skill with no fusion path and 4★+ depth is eligible for Unique promotion.

Rules

  • You must hold all prerequisite components in your tree before a fusion becomes available.
  • The fused skill is added to your tree; the component skills remain — they are not consumed.
  • Fusion is not free-form: only registry-defined fusion paths are valid. You cannot fuse arbitrary skills.
  • The registry defines which combinations are meaningful. To propose a new fusion path, open an issue or PR.
  • Basic → Unique is a one-way promotion, not a fusion — no other skills are consumed.
fuse via CLI
# See what fusion candidates you have
gaia tree    # fusion candidates appear in the output

# Fuse interactively (Gaia shows eligible combinations)
gaia fuse

# Fuse a specific target
gaia fuse autonomous-research-agent

Named Skills

A Named Skill is a canonical skill that has been claimed by a real contributor with Class C evidence or better. At the moment of naming, the skill reaches 2★ Named rank. The contributor's name attaches permanently as the Origin Contributor and renders in honor red across the registry.

Named Skills are the human layer of the registry — they answer not just "does this capability exist?" but "who has demonstrated it, where, and how well?" Multiple contributors can hold named implementations of the same canonical skill; only the first is the Origin.

1
Run gaia scan inside a Git repo containing agent skill files (SKILL.md, .claude/skills/, etc.). The scanner detects capability evidence and writes promotion candidates.
2
Run gaia push --dry-run to preview the skill batch, then gaia push to submit it for intake review via a GitHub issue.
3
A Verifier reviews the intake. If approved, the skill is added to registry/named-skills.json at 2★ with your name as Origin Contributor. You can also claim an existing unnamed canonical skill using gaia propose <skillId>.
4
Install the skill for use in your agent environment: gaia skills install contributor/skill-name --global. The skill appears under ~/.claude/skills/ (or your IDE's skill directory).
5
Continue adding evidence (gaia dev evidence) to rank the skill up from 2★ Named through Evolved, Hardened, and eventually Transcendent. At 4★ the holder gains Verifier authorization to review others' submissions.

Generic / Starless references

The canonical graph contains two kinds of nodes: generic (starless) nodes and named nodes. Understanding the difference is critical when reading gaia lookup output or the Hunter's Atlas.

A generic (starless) reference is a taxonomy node that carries no stars of its own. It represents the capability in the abstract — "web search as a category." Stars belong only to the named implementations (the skill's "children"). The generic node's effective rank is the highest star among all its named children.

In the UI, generic nodes render in italic greyed-out styling to distinguish them from named skills. The term generic is the technical descriptor; starless is the brand / collective noun.

generic (starless)
web-search
No stars of its own. Effective rank = top named child.
named implementation
karpathy /web-search 3★
Holds its own stars. Name in honor red = Origin Contributor.

Local-first design

All player-facing commands default to showing your data, not the canonical registry. gaia tree shows your unlocked skills. gaia scan looks at your local directories. gaia stats reports on your personal tree and installed skills.

To view the canonical registry instead, pass --canon to any command. This toggle is available everywhere: gaia tree --canon, gaia stats --canon, gaia graph --canon.

local-first vs canon
# Your personal tree (local-first default)
gaia tree

# The full canonical registry tree
gaia tree --canon

# Your local stats vs the full registry
gaia stats
gaia stats --canon
Local-first is intentional. The registry is a shared, global artifact. Your skill tree is yours. Commands without --canon work against the local state so that scanning, promoting, and tree-viewing are fast and private until you explicitly push for review.
