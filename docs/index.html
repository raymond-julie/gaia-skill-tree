What is Gaia?

Think of it as a Pokédex for AI agent abilities — but open, evidence-backed, and community-owned.

○
Basic
Basic Skills
The genome of every agent. Primitive, indivisible capabilities — Tokenize, Retrieve, Classify, Embed Text.
◇
Extra Skill
Extra Skills
Emerge when 2+ basics combine — RAG Pipeline, Autonomous Debug, Research. More than the sum of their parts.
◆
Ultimate Skill
Ultimate Skills
High-complexity emergent capabilities found in <1% of agents — Autonomous Research Agent, Full-Stack Developer.

The Rank System

Skills level up through evidence, not declaration.

LevelRankEvidenceWhat it means
0 BasicNone Universal LLM primitive — any capable model does this by default
I AwakenedNone Catalogued and available — the foundation tier
II NamedClass C First confirmed demonstration by a real contributor
III EvolvedClass B Reproducible, fully documented
IV HardenedClass B or A Failure modes known; battle-tested
V TranscendentClass B or A Composable and self-improving
VI ★ Transcendent ★Class A Apex — peer-reviewed, named to the agent who unlocked it

Get Started

Five steps from zero to your first skill submission.

1

Clone the registry

Grab your local copy of the canonical skill graph.

git clone https://github.com/mbtiongson1/gaia-skill-tree.git
cd gaia-skill-tree
2

Install the CLI

Puts the gaia command on your PATH with semantic search included.

pip install -e ".[embeddings]"
gaia embed
💡Run gaia embed once after install (~30 sec). Skip the [embeddings] extra if you don't need semantic search — plain pip install -e . works fine.
pip install -e ".[embeddings]"
gaia embed
🪟If gaia isn't recognized after install, run the command below in PowerShell, then restart your terminal.
$env:PATH += ";" + (python -c "import sysconfig; print(sysconfig.get_path('scripts', 'nt_user'))")
3

Initialize your project

Inside the repo you want Gaia to track, run gaia init. It auto-detects your GitHub username and skill paths, then writes a .gaia/config.json.

gaia init
💡Username is read from git remote / git config. Scan paths are auto-detected from common locations (.claude/skills/, AGENTS.md, etc.). Pass --user your-username to override.
4

Scan your repo

Gaia scans your configured paths and reports which canonical skills your agent demonstrates, plus any fusion candidates.

gaia scan
5

Push your skills for review

Creates an intake record and opens a GitHub PR against the canonical graph for maintainer review.

gaia push
💡Run gaia push --dry-run first to preview the exact JSON that would be submitted — no files written, no PR opened.

Install a named skill

Browse and install real contributor implementations directly into your repo. Use --list to see what's available, then install by ID.

gaia install --list
gaia install contributor/skill-name
gaia sync
💡Browse real skill IDs in the Named Skills Explorer below. Run gaia sync periodically to pull the latest versions of all installed skills.
★

Name your awakened skill

Once a maintainer marks your pushed skill as awakened, claim it under your GitHub username to make it installable by others.

gaia name intake/skill-batches/batch-abc.json 0 yourname/my-skill
💡Syntax: gaia name <batch-file> <skill-index> <contributor/skill-name>. The skill-name becomes your permanent ID in the registry.
◆

Contribute at the Ultimate tier

Five Ultimate skills in the registry are unclaimed — no contributor has built them yet. Build an implementation, push it through intake review, then name it to become the origin contributor. New proposals require ≥3 prerequisites, ≥3 Class A/B evidence sources, and 2 Core Maintainer approvals.

python scripts/validate.py
💡Find unclaimed slots with [Unclaimed ✦] in registry.md. Use the new_ultimate_skill PR template. Set "status": "provisional" — maintainers flip it to validated at merge.

Quick Glossary

Terms you'll see throughout the registry.

gaia.json
The canonical source of truth. Every skill lives here.
intake/
Staging area for proposed skills before they enter the graph.
graph/named/
Real implementations, attributed to contributors (e.g. karpathy/autoresearch).
prerequisites
Skills that must be present before an extra or ultimate skill can emerge.
fusion
When a set of prerequisites unlocks a higher-tier extra or ultimate skill.
evidence class
A–C quality tiers for skill demonstrations. Class A = peer-reviewed.
origin: true
Marks the first contributor to publish a named implementation of a skill.
gaia embed
Pre-computes local semantic vectors so `gaia search` works by meaning.
[Unclaimed ✦]
An Ultimate skill with a hidden seed title but no contributor implementation yet — open for the taking.
