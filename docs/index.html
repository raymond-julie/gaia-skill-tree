What is Gaia?

Think of it as a Pokédex for AI agent abilities — but open, evidence-backed, and community-owned.

○
Intrinsic
Atomic Skills
The genome of every agent. Primitive, indivisible capabilities — Tokenize, Retrieve, Classify, Embed Text.
◇
Extra Skill
Composite Skills
Emerge when 2+ atomics combine — RAG Pipeline, Autonomous Debug, Research. More than the sum of their parts.
◆
Ultimate Skill
Legendary Skills
High-complexity emergent capabilities found in <1% of agents — Autonomous Research Agent, Full-Stack Developer.

The Rank System

Skills level up through evidence, not declaration.

LevelRankEvidenceWhat it means
0 BasicNone Universal LLM primitive — any capable model does this by default
I AwakenedNone Catalogued and available — the foundation tier
II NamedClass C First confirmed demonstration by a real contributor
III EvolvedClass B Reproducible, fully documented
IV HardenedClass B or A Failure modes known; battle-tested
V TranscendentClass B or A Composable and self-improving
VI ★ Transcendent ★Class A Apex — peer-reviewed, named to the agent who unlocked it

Get Started

Five steps from zero to your first skill scan.

1

Clone the registry

Grab your local copy of the canonical skill graph.

git clone https://github.com/mbtiongson1/gaia-skill-tree.git
cd gaia-skill-tree
2

Install the CLI

One pip install puts the gaia command on your PATH.

# Core install
pip install -e .

# Strongly recommended — enables semantic search
pip install -e ".[embeddings]"
gaia embed          # ~30 sec, run once
3

Initialize your project

Inside the repo you want Gaia to track, run gaia init. It creates a .gaia/config.json config file.

gaia init
# Edit .gaia/config.json — set your GitHub username
{
  "gaiaUser": "your-github-username",
  "scanPaths": ["src", "scripts"]
}
4

Scan your repo

Gaia scans your configured paths and reports which canonical skills your agent demonstrates, plus any fusion opportunities.

gaia scan
# Prints detected skills and combination candidates
5

Submit a batch for review

Preview what would be submitted with --dry-run, then push to create an intake record.

gaia push --dry-run   # preview the JSON
gaia push             # write intake + open a PR
💡 Tip — semantic search

Once embeddings are generated, gaia search "autonomous research" finds skills by meaning, not just keywords. The model runs fully locally — no API key needed.

+

Install a named skill

Other contributors publish real implementations of generic skills. Install one directly into your repo:

gaia install karpathy/autoresearch
gaia install --list          # see what's installed
gaia sync                    # pull latest versions
★

Promote your own named skill

Once your pushed skill is awakened, claim it under your GitHub username:

# gaia name <batch-file> <skill-index> <contributor/skill-name>
gaia name intake/skill-batches/batch-abc.json 0 yourname/my-skill

Quick Glossary

Terms you'll see throughout the registry.

gaia.json
The canonical source of truth. Every skill lives here.
intake/
Staging area for proposed skills before they enter the graph.
graph/named/
Real implementations, attributed to contributors (e.g. karpathy/autoresearch).
prerequisites
Skills that must be present before a composite or legendary can emerge.
fusion
When a set of prerequisites unlocks a higher-tier composite or legendary skill.
evidence class
A–C quality tiers for skill demonstrations. Class A = peer-reviewed.
origin: true
Marks the first contributor to publish a named implementation of a skill.
gaia embed
Pre-computes local semantic vectors so `gaia search` works by meaning.
