# Brainstorming: Yggdrasil 3D Shell Model (Commission Art Specification)

This document outlines the requirements and design ideas for the 3D model shell representing Yggdrasil, the World Tree, which will sit atop the interactive DAG nodes in the Gaia Skill Tree homepage hero.

## Background & Problem Statement

Currently, the homepage features a 2D/3D dynamic projection of the DAG nodes representing AI skills. While the layout is structured botanically (root flare, centered trunk, rounded crown, side groves, and seed bulb), the connection lines and nodes feel "raw" and purely mathematical.

To elevate the visual identity to a true sacred-art representation, we want to overlay a **golden tree of light shell** modeled after Yggdrasil. The nodes of the skill tree will map to or nest within the branches and roots of this 3D light model.

## Core Design Goals

1. **Botanical Legibility**: A recognizable silhouette of a great world tree (roots, massive trunk, sprawling canopy/branches) that matches the layout of the actual node DAG.
2. **Golden Light Shell**: The model should feel like it is woven from light—semi-transparent, glowing, golden threads or meshes that do not obscure the node elements.
3. **Adaptive Scaling / 3D Layout Alignment**: The 3D model must align with the coordinates generated by `world-tree-layout.js` or provide a mapping mechanism where specific coordinates scale to fit the major branch endpoints.
4. **Commission Art Deliverable**: We need to define a standardized JSON representation for this 3D model that can be easily loaded via Three.js (e.g., as custom line/spline paths, custom vertex/face geometry, or a light-weight OBJ/GLTF to JSON export).

## Proposed Technical Approach

- **Model Specification**: A custom JSON format or lightweight GLTF/GLB structure detailing the branch/spline paths of the Yggdrasil shell.
- **Three.js Integration**: Rendered as a custom neon/glow line structure or custom shader-based geometry in `docs/js/skill-graph.js`.
- **Node Anchoring**: Mapping specific key parent/child nodes to physical branch intersections on the Yggdrasil shell.

## Next Steps

1. Brainstorm with the user on the visual aesthetic (wireframe, glowing particle tree, low-poly solid gold, or high-fidelity generative curves).
2. Refine the JSON schema/contract for the 3D model so the commissioned artist knows exactly what data format to output.
3. Prototype the loader and rendering pipeline within the existing Three.js canvas.
