Split into W2a (schema) and W2b (pipeline) because the schema decisions ripple into Sprint E — never let the pipeline commit before the schema is spec'd.

#### W2a — Benchmark evidence schema (~$4, Opus max)

**Files/paths:**
- `registry/schema/evidence/benchmark-result.schema.json` (NEW — extension of the existing `benchmark-result` evidence type — that string already exists in `src/gaia_cli/commands/dev/evidence.py`; this workstream promotes it to a real schema)

- `docs/api/v1/benchmarks/index.json` — list of live benchmarks (schema)
- `docs/api/v1/benchmarks/humaneval.json` — per-benchmark leaderboard data schema

**Schema invariants (Splurge — get right first time, will echo into Sprint E):**
- Every benchmark-result row MUST have: `benchmarkId` (semver-ish `<name>@<version>`), `score`, `unit`, `runAt` (ISO 8601), `provenance` (one of `verifier-attested` | `ci-reproduced` — never `self-attested`), `attestor` (verifier handle OR CI workflow URL + commit SHA), `datasetHash` (SHA-256 of the dataset used).
- `benchmarkInputHash` field mirrors the `<index>InputHash` pattern from v4 §Indices — reproducibility fingerprint.

**Model tier + effort:** Opus max (schema decisions are load-bearing per v4 §Splurge vs Satisfice).

**Est. token budget:** ~$4.

**Dependencies:** none. W2a can start Day 1 (in parallel with W1 scaffolding). W2b + W4 block on W2a merging.

**Migration notes:**
- **Portable:** the schema JSON file survives Sprint F unchanged. The evidence writer in `src/gaia_cli/commands/dev/evidence.py` survives (Python CLI is portable per v4).
- **Rewrites:** none for this workstream.
- **Invariants:** schema field names + shape are frozen once merged; downstream index consumers depend on them. This is a schema-hardening PR — treat as API contract.

#### W2b — Benchmark #1 pipeline (~$5, Opus high)

**Files/paths:**
- `scripts/benchmarks/humaneval/run.py` — CI-reproducible harness
- `scripts/benchmarks/humaneval/dataset.jsonl` (or download-and-hash step in CI)
- `scripts/benchmarks/humaneval/verify.py` — reads a submitted score, checks provenance
- `.github/workflows/benchmark-humaneval-ci.yml` — reproduces the benchmark on submitted commits, writes evidence directly
- `src/gaia_cli/commands/push.py` — add `--benchmark <name> --score <pct>` flags; wire through to evidence writer
- `src/gaia_cli/commands/dev/evidence.py` — validate `benchmark-result` type against W2a schema
- `docs/named/**/skill.md` (one skill) — evidence entry added via `gaia push --benchmark humaneval --score X` (proves KC4 end-to-end)

**Verifier / CI reproduction gate (MANDATORY):**
- Two acceptable provenance paths:
  - **(a) Verifier attestation** — a 4★+ verifier signs off in `docs/verifier-signoffs/YYYY-MM/humaneval-<skill>.md` (existing pattern — extend `scripts/check_verifier_signoffs.py`)
  - **(b) CI reproduction** — `benchmark-humaneval-ci.yml` runs on the submitted commit SHA, writes evidence with `provenance: ci-reproduced`, `attestor: <workflow-url>@<commit>`
- **Never accept self-attested benchmark-result evidence.** `gaia push --benchmark ... --score ...` writes a **pending** row (`provenance: pending`) that CI or a verifier must promote to `verifier-attested`/`ci-reproduced` before it counts. The validator (extend `scripts/validate_redaction.py` sibling `validate_benchmarks.py`) rejects merges with `pending` benchmark rows on `main`.

**Model tier + effort:** Opus high (pipeline touches CLI + CI + evidence writer).

**Est. token budget:** ~$5.

**Dependencies:** W2a merged. Blocks W4.

**Migration notes:**
- **Portable:** `scripts/benchmarks/humaneval/**`, the CLI push flags, the CI workflow. All Python + YAML — survives Sprint F.
- **Rewrites:** none (Skill Tree keeps the CLI + registry per v4 Sprint F plan).
- **Invariants:** CLI verb signature `gaia push --benchmark <name> --score <pct>` frozen at v6.1.0; changing it is a v7 major bump.

## 4. Publish gate for Content Engine (first 4 weeks)

**Location:** environment variable `GAIA_CONTENT_ENGINE_PUBLISH` (set in `.github/workflows/weekly-content-engine.yml` as a workflow-level env var pulled from a GitHub Environment named `content-engine-live`).

**Semantics:**
- Unset or `0` → weekly report generates into `docs/reports/DRAFT/YYYY-WW.md` (gitignored — never leaves the runner). A GitHub Actions summary artifact posts the draft for review. No PR opens.
- `1` → weekly report writes to `docs/reports/YYYY-WW/index.html`, opens an auto-PR against `main` with body `Resolves` nothing (this is an add-only content commit), applies label `content-engine`, requests review from Marco.

**Who flips the flag:** Marco, weekly, in the GitHub Environments UI (`Settings → Environments → content-engine-live → environment secrets`). Toggle happens on Monday evening after he reviews the previous week's artifact.

**Escalation path if a bad report ships:**
1. Marco reverts the merge commit on `main` (single revert PR).
2. Flip `GAIA_CONTENT_ENGINE_PUBLISH` back to `0`.
3. File an issue tagged `content-engine` + `bug` with the salvage-layer flag (L1/L2/L3-mechanical) from the report frontmatter — this identifies whether the failure was L1 quality or a fallback assembly artifact.
4. Fix, then flip back to `1` the next Monday.

**Sunset:** after 4 consecutive clean weeks (per v4 §Content Engine), Marco may flip the default to `1` and remove the environment gate. Track in `founder/MEMORY.md`.

## 5. Verifier attestation OR CI reproduction — benchmark evidence

Two acceptable provenance paths for every `benchmark-result` evidence row:

| Path | Trigger | Attestor field | Enforcement |
|---|---|---|---|
| (a) Verifier attestation | 4★+ verifier signs off in `docs/verifier-signoffs/YYYY-MM/<benchmark>-<skill>.md` | verifier handle | `scripts/check_verifier_signoffs.py` extension |
| (b) CI reproduction | `.github/workflows/benchmark-<name>-ci.yml` runs the harness on submitted SHA | `<workflow-url>@<commit-sha>` | Workflow writes evidence directly; no manual push required |

**Explicitly rejected:**
- Self-attested benchmark scores (`gaia push --benchmark ... --score ...` without CI or verifier follow-up) — the row lands as `provenance: pending` and is stripped by `scripts/validate_benchmarks.py` on the pre-merge CI. Never counts toward TM. Never appears on leaderboards.
- Mirrored scores (W3) — accepted as citations only. Distinct badge. Excluded from TM contribution.

**Rationale (v4 §Content Engine section):** the first bad or inflated benchmark permanently damages the megaphone's credibility. Verifier/CI gate is the single most load-bearing invariant of Sprint D.
