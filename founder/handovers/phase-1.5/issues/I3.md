## Definition of Done

- [ ] `scripts/migrateTrustMagnitude.py` runs idempotently. For every skill in `registry/named/` and `registry/nodes/`:
  1. `enforceAntiAutoMint()` strips phantom rows.
  2. `computeTrustMagnitude()` derives the new aggregate.
  3. Writes `trustMagnitude`, `overallTrustGrade`, `apexGateStatus`, `verification.tier`, `verification.firstEvidenceAt` (if missing — set to first `evidence_added` event in timeline), `provisional` flag (any A/S row missing `sourceStartedAt`).
  4. Appends a `migrate_trust_magnitude` timeline event with before/after summary in `details`.
- [ ] `registry/named-skills.json` regenerated by `scripts/generateNamedIndex.py`.
- [ ] `docs/meta/JUN_2026_TRUST_REGRADE.md` written + rendered to HTML by `gaia-post`. Section order per delta + Marco's 2026-06-16 directive (apex demotions LEAD).
- [ ] Second run produces empty diff (idempotency).

## Critical constraints

1. **Skill-tree files off-limits.** Touches `registry/named/` and `registry/nodes/` only. Does NOT modify `level` (that's I5), `evidence[]` row grades, slot data, or any `skill-trees/<username>/skill-tree.json`.
2. **Idempotent.** Use stable hash of `(skillId, evidence[].url, evidence[].type, evidence[].grade)` as input; skip if hash matches `trustMagnitudeInputHash`.
3. **Direct file edit fallback.** If `gaia dev` does not yet support writing `trustMagnitude`, do direct frontmatter edits + log `(direct edit — CLI gap)` per `CLAUDE.md`. Open follow-up issue for `gaia dev migrate-trust-magnitude` verb.
4. **Apex demotions deferred to I5.** This script computes `apexGateStatus` so the demotion list is auditable, but the actual `level: 6★ → 5★` is I5's job.
5. **`sourceStartedAt` provisional flag** (delta §C-1): any A/S row missing this field gets `provisional: true` set on the parent skill + a row in the stamp report's "Provisional grades" section.
