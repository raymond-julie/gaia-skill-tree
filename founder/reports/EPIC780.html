Dynamic Commands

1. CLI Dispatch Leverage Point

Instead of a massive dispatcher where interface and implementation are tightly coupled inside main.py, we introduce the Command Protocol: 

class Command(Protocol):
    name: str
    description: str
    needs_auth: bool = False
    
    def configure(self, parser: ArgumentParser) -> None: ...
    def execute(self, args: Namespace) -> int: ...

This establishes a rigid, thin interface. Command implementations are auto-discovered dynamically from src/gaia_cli/commands/. Adding a CLI feature is now as simple as adding a file containing a COMMAND matching the protocol.

2. Artifact Pipeline Seam

Merge conflicts on generated indices (like registry/gaia.json) are eliminated by introducing an artifact seam:

  • Developers use local gaia dev build to write artifacts (untracked).
  • GitHub Action workflow builds and packages these assets.
  • Assets are attached directly to GitHub Releases, bypass-committing directly to the branch.
