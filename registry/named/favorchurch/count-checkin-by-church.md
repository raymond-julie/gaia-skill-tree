# Count Check-in by Church (live, normalized)

## Overview

The conference **live-count** Google Sheet (feeds `favor.church/conferencelivecount`) is refreshed by
hand: export attendees → paste values into `MASTERLIST!A4`. This skill adds two tabs that turn the raw
paste into an accurate, **self-updating** per-church check-in count:

- **`CHECK-IN BY CHURCH`** — a live `MAP`/`LAMBDA` + `QUERY` table: one row per church, with how many
  attendees checked in ("Yes"), counted **once per normalized church** (spelling variants folded), plus a
  summary block. Recomputes automatically every time MASTERLIST is re-pasted.
- **`CHURCH LOOKUP`** — a flat `nkey → "Name | City"` map (semi-static) that drives the normalization.

It also repoints **`SUMMARY!C14`** (the public "unique churches checked in" number) to the correct
normalized count.

**Core principle:** normalization is judgment, and that judgment already lives in the
[`update-unique-churches`](../update-unique-churches/SKILL.md) sheet. This skill *reuses* that sheet's
normalized names + "Merged variants" as the single source of truth — it never re-invents matching.

---

## The bug this fixes

`SUMMARY!C14` originally was:

```
=COUNTUNIQUEIFS(MASTERLIST!N5:N, MASTERLIST!B5:B,"Yes", MASTERLIST!N5:N,"<>", MASTERLIST!BA5:BA,"completed")
```

It is wrong in **both** directions:

1. **Over-counts** — it counts distinct *raw text*, so one church appears many times ("Crossroads Church
   Cebu" + "Crossroads Church, Cebu" + "Crossroads Church - Cebu City" … are all one church). Measured:
   **256 raw vs ~157 normalized.**
2. **Under-counts** — it reads only column **N**. ~⅓ of non-Favor check-ins put their church only in the
   fallback columns **S / T / AH / AI**; C14 missed every one.
3. Counts junk ("Others", personal names, "N/A") as churches.

The `CHECK-IN BY CHURCH` tab fixes all three; `C14` then points at its unique count.

## Workflow

Work in a scratch dir. Script: `build_lookup.py` (in this folder). All reads/writes via the Composio CLI.

### A. First-time setup (create the tabs)

Skip to **B. Refresh** if the tabs already exist.

**1. Build the lookup from the church-count sheet.**
```bash
CC=1PT2e_9nk9gmlPKcGQloXmTLgZZt88MLzyt0kL249oFE
composio execute GOOGLESHEETS_VALUES_GET -d "{\"spreadsheet_id\":\"$CC\",\"range\":\"Church Count (Normalized)!A4:E400\"}" > cc.json
python3 build_lookup.py cc.json lookup.json      # -> [[nkey,"Name | City"], ...], header row included
```

**2. Create the two tabs** (`GOOGLESHEETS_ADD_SHEET`, ~6200 rows / 12 cols each), titles
`CHURCH LOOKUP` and `CHECK-IN BY CHURCH`.

**3. Write the lookup** to `CHURCH LOOKUP!A1` (value_input_option `RAW`).

**4. Write the `CHECK-IN BY CHURCH` formulas** (value_input_option `USER_ENTERED`):

- `A1` title, `A2` note, `A3` = `["Church | City","Checked In"]`.
- `A4` (the per-church table):
  ```
  =IFERROR(QUERY(J2:J,"select Col1, count(Col1) where Col1 is not null and Col1 <> '' group by Col1 order by count(Col1) desc label Col1 'Church | City', count(Col1) 'Checked In'"),"")
  ```
- `J1` note, `J2` (the live helper — normalized church per checked-in row):
  ```
  =MAP(MASTERLIST!B5:B,MASTERLIST!M5:M,MASTERLIST!N5:N,MASTERLIST!S5:S,MASTERLIST!T5:T,MASTERLIST!AH5:AH,MASTERLIST!AI5:AI,MASTERLIST!BA5:BA,
   LAMBDA(b,m,n,s,t,ah,ai,st,
    IF((st="completed")*(b="Yes")*(LEFT(LOWER(m&""),5)<>"favor")=1,
     LET(v,LAMBDA(x,(TRIM(x&"")<>"")*(LEFT(LOWER(x&""),5)<>"favor")*(NOT(REGEXMATCH(LOWER(TRIM(x&"")),"^(n/?a|none\.?|wala|n|x|n/q)$")))),
      pick,IF(v(n)=1,n,IF(v(s)=1,s,IF(v(t)=1,t,IF(v(ah)=1,ah,IF(v(ai)=1,ai,""))))),
      IF(pick="","",IFERROR(VLOOKUP(LOWER(REGEXREPLACE(pick&"","[^A-Za-z0-9]","")),'CHURCH LOOKUP'!$A:$B,2,FALSE),"⚠ "&pick))),
     "")))
  ```
- Summary cells `D1:E5`:
  | Cell | Label (D) | Formula (E) |
  |---|---|---|
  | 1 | Unique churches checked in: | `=IFERROR(COUNTUNIQUE(FILTER(J2:J,(J2:J<>"")*(NOT(REGEXMATCH(J2:J&"","^⚠"))))),0)` |
  | 2 | Matched check-ins: | `=IFERROR(COUNTA(FILTER(J2:J,(J2:J<>"")*(NOT(REGEXMATCH(J2:J&"","^⚠"))))),0)` |
  | 3 | Unmatched (⚠ fold later): | `=SUMPRODUCT(--REGEXMATCH(J2:J&"","^⚠"))` |
  | 4 | Favor-location check-ins: | `=COUNTIFS(MASTERLIST!BA5:BA,"completed",MASTERLIST!B5:B,"Yes",MASTERLIST!M5:M,"Favor*")` |
  | 5 | Total check-ins: | `=COUNTIFS(MASTERLIST!BA5:BA,"completed",MASTERLIST!B5:B,"Yes")` |
  Stash the previous `C14` formula in `G1:G2` for reference.

**5. Repoint `SUMMARY!C14`** → `='CHECK-IN BY CHURCH'!E1` (value_input_option `USER_ENTERED`).

### B. Refresh (keep the lookup in sync)

The `CHECK-IN BY CHURCH` tab is **live** — it recomputes on its own every time MASTERLIST is re-pasted.
The only thing that goes stale is the `CHURCH LOOKUP` map, when **new churches** appear. To refresh:

1. Run [`update-unique-churches`](../update-unique-churches/SKILL.md) on the church-count sheet so any new
   churches (including ones showing as `⚠` here) get folded into its normalized list + variants.
2. Re-run **A.1** (rebuild `lookup.json`) and overwrite `CHURCH LOOKUP!A1:B…`. If the new map is shorter,
   clear the trailing rows.
3. Done — `CHECK-IN BY CHURCH` and `SUMMARY!C14` recompute automatically.

## Verification

After writing, read back `CHECK-IN BY CHURCH!D1:E5` and confirm against an independent tally
(`build_lookup.py`'s matching logic mirrors the sheet's `nkey` + priority). Spot-check that `SUMMARY!C14`
equals `E1`, and that the top rows look right (Crossroads/Cebu, Chosen by God/Malolos, New Life… should top
the list). If `E1` is far below the raw distinct-N count, that's expected — variants are being folded.

## Gotchas

- **`⚠` rows are a review flag, not the count.** They're raw strings not in the lookup — mostly junk
  (people who typed their own name in the church field) plus a few genuinely new churches. They are
  **excluded** from the `E1` unique count. To clear real ones, refresh (section B).
- **The numbers move during the event.** It's a live count — total check-ins climb as people scan in. The
  formulas track the current MASTERLIST automatically; don't mistake a jump for an error.
- **Pipe-delimiter strings** like `Word For All Nation Inc. | Quezon City` re-surface as `⚠` because ` | `
  is the church-count "Merged variants" delimiter (they split on re-read). Harmless here; they're a known
  quirk of the source sheet.
- **Fixed column letters.** The helper hardcodes `N/S/T/AH/AI/M/B/BA`. If the export layout drifts, update
  the `MAP` ranges (they're header-detected in the church-count skill, but live formulas can't self-detect).
- **Favor-location attendees** (col `M` starts with "Favor") are intentionally excluded from the per-church
  list — they're counted separately in `E4`. The list is non-Favor churches only, matching the church-count
  sheet's scope.
