## Overview

Turns ambiguous stakeholder inputs into structured, traceable requirements. Produces BRDs, user stories with acceptance criteria, and traceability matrices that map requirements to implementation tasks.

## Key behaviours

- Elicits requirements through structured questioning and stakeholder interviews
- Writes user stories in Given/When/Then and acceptance criteria format
- Maintains a requirements traceability matrix linking specs to test cases
- Flags scope creep and conflicting requirements with resolution proposals
