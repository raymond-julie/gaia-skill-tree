## Description
Systematically stress-tests a plan or design through relentless, one-question-at-a-time Socratic questioning. Walks the decision tree branch-by-branch, resolving concept dependencies, surfacing contradictions against the existing codebase, and crystallizing ubiquitous language — optionally persisting resolved decisions to CONTEXT.md and ADRs.

## Fusion Condition
Enforces a strict questioning loop that prevents proceeding to implementation until all branches are resolved. Requires domain-model integration (CONTEXT.md) for higher-order alignment.
